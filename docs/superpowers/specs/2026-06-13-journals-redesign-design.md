## Goal

Replace the single "Journal Entries" page with a dedicated "Journals" sidebar section containing four specialized journal views. All roles can view journals; write actions are restricted to accountant/admin.

## Architecture

New `app/journals/` blueprint owns four filtered list views. The existing `journal_entries` blueprint retains all CRUD (create, edit, post, cancel, delete) â€” nothing moves. The journals blueprint queries `JournalEntry` with different `entry_type` filters per view. No new database columns required.

### Accounts Payable Journal â€” `/journals/ap`

## JV Numbering

Add `generate_jv_number(branch_id)` to `app/journal_entries/utils.py`. Format: `JV-YYYY-MM-NNNN`
(year + zero-padded month + 4-digit sequence, resets each month). Queries `JournalEntry` filtering
`entry_number.like('JV-%')` for the current year-month and branch. The `create` view in
`journal_entries/views.py` calls `generate_jv_number()` instead of `generate_entry_number()` to
pre-populate the entry_number field on GET. Existing entries with `JE-` prefix continue to display correctly.
