# Customer Withholding-Tax List (Customer↔WHT parity with Vendor) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give `Customer` a many-to-many withholding-tax list (mirroring `Vendor`), so the Sales Invoice **and** Cash Receipt line-WT dropdowns scope to the customer's WHT codes — locked "No withholding tax for this customer" when none, auto-selected when one — exactly like APV's vendor-scoped WT dropdown.

**Architecture:** Add a `customer_withholding_taxes` association table + `Customer.withholding_taxes` relationship (direct copy of `vendor_withholding_taxes`). The customer form/quick-add gain a WHT checkbox multi-select. `/customers/<id>/defaults` returns the M2M list. SI and CRV forms replace their global `whtCodes`/`all_whts` line scoping with the customer's list, reusing APV's `buildWhtOptions`/`rebuildAllWhtSelects` pattern. The legacy single `default_wt_code` column is **kept** (still in form/exports/audit) — no column drop.

**Tech Stack:** Flask, SQLAlchemy, Flask-Migrate/Alembic, WTForms, Choices.js, pytest, Playwright.

## Global Constraints

- Keep `Customer.default_wt_code` — **no column drop, no data migration** (decision: keep).
- Scope **both** SI and CRV line-WT to the customer's list (decision: include CRV).
- All accounting/UI work done as a **non-admin** during manual verification; admin only for admin-only actions.
- No JS popups — toasts use the existing `showToast` helper; no `confirm/alert/prompt`.
- Model edits require this plan's sign-off (already given); run `flask db migrate` + `flask db upgrade`.
- After editing any `app/static/*` asset, bump its `?v=N` on every template that links it. (The SI/CRV scoping JS is inline in the form templates — no static bump needed there.)
- Reference template: **Vendor** — `app/vendors/models.py:9-12,51-53`, `app/vendors/views.py:164-168,244-247`, `app/accounts_payable/templates/accounts_payable/form.html` (`buildWhtOptions` :350, `rebuildAllWhtSelects` :365, vendor change-handler :867).
